# 系统分步开发路线

## 第 1 步：基础工程稳定化

目标：

- 保持当前 Spring Boot + Vue MVP 可运行
- 统一异常处理
- 明确数据库表结构

当前状态：

- 已完成

## 第 2 步：接入 MySQL 持久化

目标：

- 把首页分类、服务人员、订单从内存数据迁移到数据库
- 引入实体类、Repository/Mapper、初始化数据

输出：

- 可读取 MySQL 的后端版本

## 第 3 步：多角色登录与权限控制

目标：

- 普通用户登录
- 服务人员申请入驻
- 管理员登录与审核

输出：

- 用户、服务人员、管理员三角色闭环

## 第 4 步：订单与履约增强

目标：

- 完善订单状态流转
- 服务进度上报
- 取消、售后、评价反馈

输出：

- 更完整的交易履约链路

## 第 5 步：后台运营能力

目标：

- 服务项目管理
- 服务人员审核
- 订单监管
- 数据统计

输出：

- 毕业设计答辩级别的完整管理后台

## 下一步建议

下一轮最值得继续做的是：

- 第 2 步：接入 MySQL 持久化

因为只要数据库一接上，后面的登录、审核、订单流转才真正有“系统”基础。
