# 基于任务书的 Taskrabbit 风格家政服务预约平台落地说明

## 1. 目标对齐

任务书主题为《基于 Java Web 的家政服务预约平台的设计与实现》，明确要求使用：

- 前端：`HTML + CSS + JavaScript + Vue`
- 后端：`Spring Boot + Java`
- 数据库：`MySQL`

结合任务书与 Taskrabbit 的产品模式，本项目建议落地为：

- 参考 Taskrabbit 的“服务浏览 -> 选择服务人员 -> 预约下单 -> 跟踪进度 -> 评价反馈”链路
- 业务内容改造为任务书规定的家政场景
- 面向毕业设计，先交付“可演示的 MVP 原型 + 可继续扩展的工程骨架”

## 2. 任务书功能与当前实现映射

### 已落地的 MVP 演示链路

- 首页：展示平台定位、热门服务、平台优势、推荐服务人员
- 家政服务浏览：按服务类型筛选服务人员
- 服务人员详情：展示资质、经验、服务区域与案例
- 在线预约下单：填写时间、地址、需求说明并创建订单
- 订单进度跟踪：查看待服务、服务中等状态
- 数据统计：管理员看板展示总订单、满意度和各服务类型销量

### 当前仍是演示型实现

- 多角色登录与权限控制：未接 JWT / 数据库
- 服务人员入驻申请与审核：未实现完整审批流
- 在线支付：暂未对接微信支付/支付宝
- 服务过程图片上传：暂未接文件存储
- 售后反馈、评价留言：暂未实现表单与后台处理
- 数据库存储：当前后端使用内存模拟数据

## 3. 当前工程结构

```text
housekeeping/
  |- backend/
  |   |- housekeeping-server/
  |- frontend/
  |   |- web/
  |- doc/
```

## 4. 后续最推荐的迭代顺序

1. 接入 MySQL，将内存数据迁移为实体类、Mapper/Repository 与 SQL 表
2. 增加登录、JWT、用户/服务人员/管理员三角色权限
3. 实现服务人员申请入驻与管理员审核
4. 完善订单状态机、取消规则、售后反馈与评价模块
5. 接入图片上传、支付回调和消息通知

## 5. 本次实现的意义

这次不是只写方案，而是先把一个“能演示任务书核心流程”的前后端 MVP 直接搭起来。这样你后面继续做论文、答辩演示、功能扩展时会轻松很多，因为项目已经具备了：

- 清晰的页面结构
- 可运行的 Spring Boot + Vue 工程骨架
- 与任务书一致的功能主链路
- 与 Taskrabbit 接近的产品交互方式
